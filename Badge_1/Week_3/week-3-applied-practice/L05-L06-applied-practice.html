Week 3 Applied Practice: Blueline Annotations

Keyboard, Touch & Responsive Design Combined Project

🔍 Discover Badge 🏷️ Modules 1E + 1F 📝 28 Points

This applied practice project brings together the skills you learned in Week 3: keyboard accessibility, focus indicators, touch target sizing, and responsive design requirements. These skills ensure your designs work for users who navigate by keyboard, rely on visible focus states, need larger touch targets, or use zoom and text spacing adjustments.

You will receive a design mockup and create a complete set of accessibility annotations covering focus indicator specifications, touch target measurements, and responsive behavior notes. This mirrors real-world design handoff where interaction and responsive specifications are delivered together.

🎯 What You'll Demonstrate

Keyboard & Focus (L05)

  • Identify all interactive elements that need keyboard access
  • Design focus indicators that meet WCAG 2.2 SC 2.4.11 requirements (2px perimeter, 3:1 contrast)
  • Annotate focus indicator specifications (color, thickness, contrast ratio)
  • Identify potential keyboard traps or focus order issues

Touch Targets (L05)

  • Measure touch targets using Stark's touch target checker
  • Identify targets below 24x24 pixel minimum (SC 2.5.8)
  • Annotate touch target dimensions and recommend fixes

Reflow & Responsive (L06)

  • Identify components that need flexible sizing (min-height vs fixed height)
  • Document expected reflow behavior at narrow viewports
  • Flag potential text spacing issues in fixed-height containers

📦 Assignment Instructions

What You Receive

A design mockup in Figma containing a responsive interface with multiple interactive elements requiring accessibility specifications.

📎 Duplicate Figma File to Begin

The mockup includes:

  • Navigation bar with multiple interactive elements
  • Form with various input types and buttons
  • Card grid layout (multi-column on desktop)
  • Icon buttons and text links
  • Mobile viewport version showing responsive behavior
  • Components with both fixed and flexible sizing

What You Deliver

An annotated Figma file with Stark Blueline annotations covering:

Part 1: Focus Indicator Annotations

  • All interactive elements identified with focus state specifications
  • Focus indicator color, thickness (minimum 2px), and contrast ratio (minimum 3:1)
  • Notes for any elements where default browser focus is sufficient vs custom focus needed

Part 2: Touch Target Annotations

  • All touch targets measured with dimensions noted
  • Targets below 24x24px flagged with recommended fix (padding, spacing, or resize)
  • Notes explaining any exceptions (inline links within text)

Part 3: Responsive & Flexibility Notes

  • Components requiring min-height instead of fixed height identified
  • Notes describing expected reflow behavior (e.g., "Cards stack to single column at 320px")
  • Fixed-height containers that may clip text flagged

Step-by-Step Process

  1. Identify interactive elements: Review the design and list all buttons, links, form fields, and other interactive components that need keyboard access and focus states.
  2. Design focus indicators: For each interactive element, specify a focus indicator that meets WCAG 2.2 requirements (2px minimum, 3:1 contrast). Annotate color value, thickness, and calculated contrast ratio.
  3. Measure touch targets: Use Stark's touch target checker on the mobile version. Document dimensions for all touch targets and flag any below 24x24px.
  4. Review responsive behavior: Compare desktop and mobile versions. Add notes describing how multi-column layouts should reflow to single column.
  5. Check container flexibility: Identify any components with fixed heights that could clip text when users adjust text spacing. Recommend min-height alternatives.
  6. Final review: Verify all annotations are clear enough for engineering implementation without follow-up questions.

💡 Tips for Success

Focus Indicators

  • Minimum 2px thickness: The perimeter of the focus indicator must be at least 2 CSS pixels
  • 3:1 contrast minimum: Measure contrast against adjacent colors (both the element and background)
  • Be specific: "#0066CC, 3px solid outline, 4.5:1 contrast vs white background"
  • Consider both light/dark modes: If the design has multiple themes, specify focus styles for each

Touch Targets

  • 24x24px minimum: WCAG 2.2 SC 2.5.8 requirement
  • Spacing alternative: If targets are smaller, 24px spacing between them can satisfy the requirement
  • Inline links exception: Links within paragraph text are exempt from size requirements
  • Recommend solutions: "Add 8px padding" or "Increase icon button to 44x44px for easier tapping"

Responsive & Flexibility

  • Document behavior, not breakpoints: "Cards stack to single column at narrow viewports"
  • min-height over height: Allows containers to grow when text spacing is adjusted
  • Flag fixed-height buttons: These often clip text like "Subscribe to Newsletter"
  • Look for text truncation: Components that use ellipsis may fail text spacing requirements

📜 Related WCAG Success Criteria

Keyboard & Focus

  • 2.1.1 Keyboard (Level A): All functionality operable via keyboard
  • 2.4.7 Focus Visible (Level AA): Keyboard focus indicator is visible
  • 2.4.11 Focus Appearance (Level AA, WCAG 2.2): Focus indicator meets size and contrast requirements
  • 2.5.8 Target Size Minimum (Level AA, WCAG 2.2): Touch targets are at least 24x24 pixels

Reflow & Zoom

  • 1.4.4 Resize Text (Level AA): Text can be resized to 200% without loss of content
  • 1.4.10 Reflow (Level AA): Content reflows at 400% zoom without horizontal scrolling
  • 1.4.12 Text Spacing (Level AA): No loss of content when text spacing is adjusted
