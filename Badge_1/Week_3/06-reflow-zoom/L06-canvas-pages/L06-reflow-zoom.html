📋 Learning Objectives

Learn how to design layouts that reflow at 400% zoom, create components that accommodate text spacing adjustments, and ensure designs work in both portrait and landscape orientations. These skills are essential for users with low vision, dyslexia, and anyone using mobile devices.

Learning Outcomes

By the end of this lesson, you will be able to:

ID Outcome
1F.01 Design for 400% zoom without horizontal scroll by creating layouts that reflow to single column at high zoom levels
1F.02 Ensure text spacing adjustments work by designing components that accommodate increased line height, letter spacing, and word spacing
1F.03 Apply orientation requirements by ensuring content works in both portrait and landscape orientations

Knowledge Objectives

  • Understand that SC 1.4.10 (Reflow) requires content to be usable at 400% zoom without horizontal scrolling.
  • Know that 400% zoom on a 1280px viewport creates a 320px equivalent viewport (similar to mobile).
  • Understand SC 1.4.12 (Text Spacing) requires content to remain usable when users adjust line height (1.5x), letter spacing (0.12x), word spacing (0.16x), and paragraph spacing (2x).
  • Know that fixed-height containers cause text to overflow or become clipped when text spacing is adjusted.
  • Understand that SC 1.3.4 (Orientation) requires content to work in both portrait and landscape unless a specific orientation is essential for the function.
  • Recognize legitimate orientation exceptions: piano keyboard, check deposit scanning, projector interface.

Skills Objectives

  • Design flexible layouts that stack to single column at narrow viewports.
  • Avoid fixed-width containers that prevent reflow.
  • Design components with flexible heights that expand with content (min-height instead of fixed height).
  • Test designs at 400% zoom and with text spacing adjustments using browser extensions or bookmarklets.
  • Design layouts that adapt to both portrait and landscape orientations.

Application Objectives

  • Review designs at 400% zoom to verify reflow behavior and document expected responsive behavior.
  • Identify components that would clip or overflow text when text spacing is adjusted.
  • Document flexible sizing requirements in design specs (min-height rather than fixed height).
  • Communicate responsive requirements and reflow behavior to engineering.
  • Verify designs work in portrait and landscape on target devices and document any essential orientation restrictions with justification.

📜 Related WCAG Success Criteria

  • 1.3.4 Orientation (Level AA): Content must not restrict display to a single orientation unless essential
  • 1.4.4 Resize Text (Level AA): Text must be resizable to 200% without loss of content or functionality
  • 1.4.10 Reflow (Level AA): Content must reflow at 400% zoom without horizontal scrolling
  • 1.4.12 Text Spacing (Level AA): No loss of content when users adjust line height, letter spacing, word spacing, and paragraph spacing

💡 Key Concepts

Why Reflow and Zoom Matter All

Browser zoom is more common than screen readers. When designs break at zoom, you're excluding millions of users including people with low vision (12+ million in the US), aging users whose vision changes naturally, mobile users zooming for comfort, and users with cognitive needs who find larger text reduces reading fatigue.

Key insight: When content breaks at zoom, users must scroll horizontally for every line of text. This makes reading exhausting or impossible.

The Reflow Requirement 1F.01

400% zoom on 1280px = 320px viewport

Content must reflow at 400% zoom without requiring horizontal scrolling to read text. At 400% zoom on a 1280px viewport, users see the equivalent of a 320px wide viewport, similar to a mobile phone.

Content that must reflow:

  • Body text and paragraphs
  • Navigation menus (collapse to hamburger)
  • Form fields and labels
  • Images (scale or stack)
  • Cards and content blocks
  • Sidebars (collapse or move below main content)

Exceptions (may scroll horizontally):

  • Data tables (two-dimensional by nature)
  • Complex diagrams and maps
  • Video player controls
  • Toolbars with many options

Common reflow failures:

  • Fixed-width containers: Container wider than viewport causes horizontal scroll
  • Multi-column layouts that don't stack: Columns too narrow to read
  • Absolutely positioned elements: Element overlaps or goes off-screen

Text Spacing Requirements 1F.02

Users must be able to adjust text spacing without losing content or functionality. SC 1.4.12 requires content to support these adjustments:

Property Minimum Example (16px font)
Line height 1.5x font size 24px line height
Letter spacing 0.12x font size 1.92px spacing
Word spacing 0.16x font size 2.56px spacing
Paragraph spacing 2x font size 32px paragraph gap

Users with dyslexia, low vision, or cognitive disabilities benefit from text spacing adjustments. Increased spacing prevents letters from "dancing" or merging and makes text easier to track across lines.

✗ Fixed-Height Container Problem:

A button with height: 40px clips text when spacing is adjusted. "Subscribe to Newsletter" becomes "Subscribe to Newslet..."

✓ Flexible Height Solution:

Use min-height: 40px instead. The button grows to fit text when line height increases.

Orientation Support 1F.03

SC 1.3.4 requires content to work in both portrait AND landscape unless a specific orientation is essential for the function. Users with mounted devices (wheelchairs, bed stands, kitchen mounts) cannot always rotate their device.

The test question: Would the core function break if orientation changed? If no, it's not an essential restriction.

✓ Valid Exceptions (function requires it):
  • Piano keyboard (needs horizontal space)
  • Bank check deposit (camera alignment)
  • Projector remote (matches projector)
✗ Invalid Exceptions (preference only):
  • "Looks better in portrait"
  • "Designed for landscape scrolling"
  • "Mobile version is portrait-only"

🎨 Designer's Role

Your job is not to code responsiveness, but to design for it.

Reflow behavior is primarily implemented by engineering. You don't annotate reflow in Bluelines. But you must design layouts that CAN reflow.

Your action items:

  • Design with breakpoints in mind: Show how layouts adapt at 320px
  • Avoid fixed widths in specs: Use percentage-based or flexible sizing
  • Use min-height instead of height: Components should grow with content
  • Document reflow behavior: "Sidebar collapses to hamburger menu at narrow viewports"
  • Test your prototypes: Zoom to 400% and verify the experience
