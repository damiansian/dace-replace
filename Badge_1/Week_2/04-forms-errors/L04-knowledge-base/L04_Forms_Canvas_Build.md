<h2 style="margin: 0 0 12px 0; font-size: 28px; color: #212529;">Lesson 04: Forms and Errors</h2>
<div style="display: flex; flex-wrap: wrap; gap: 14px; margin-top: 18px;"><span style="display: inline-flex; align-items: center; gap: 8px; background-color: #e6fcf5; color: #087f5b; padding: 10px 18px; border-radius: 24px; font-size: 15px;"> <span style="font-size: 18px;">🔍</span> Discover Badge </span> <span style="display: inline-flex; align-items: center; gap: 8px; background-color: #e7f5ff; color: #1864ab; padding: 10px 18px; border-radius: 24px; font-size: 15px;"> <span style="font-size: 18px;">🏷️</span> Module 1D </span></div>
</div>
<p style="font-size: 17px; color: #495057; margin-bottom: 32px; padding: 0 4px;">Forms are where users provide information, and accessibility failures here directly prevent task completion. A form without proper labels is like a paper form with no field names. A form with color-only error states leaves colorblind users guessing. In this lesson, you will learn to design clear labels, create accessible error states, and understand how autocomplete benefits users with motor and cognitive disabilities.</p>

<h4 style="margin: 0 0 12px 0; font-size: 17px; color: #212529; padding-bottom: 8px; border-bottom: 2px solid #d0ebff;">Knowledge Objectives</h4>
<ul style="margin: 0 0 28px 0; padding-left: 24px; color: #212529; font-size: 14px;">
<li style="margin-bottom: 10px;">Explain why form labels are accessible names for form controls.</li>
<li style="margin-bottom: 10px;">Describe the problems with placeholder-only labels (disappear on input, often low contrast).</li>
<li style="margin-bottom: 10px;">Identify WCAG requirements for error identification (3.3.1) and error suggestions (3.3.3).</li>
<li style="margin-bottom: 10px;">Explain how autocomplete reduces keystrokes for users with motor disabilities and memory demands for users with cognitive disabilities.</li>
<li>Recognize common form accessibility failures.</li>
</ul>
<h4 style="margin: 0 0 12px 0; font-size: 17px; color: #212529; padding-bottom: 8px; border-bottom: 2px solid #d0ebff;">Skills Objectives</h4>

<li style="margin-bottom: 10px;">Evaluate error states for text descriptions and suggested fixes.</li>
<li style="margin-bottom: 10px;">Write error messages that explain the problem AND suggest a solution.</li>
<li>Annotate tab order, labels, and error specifications in design handoffs.</li>

<h3 style="margin: 0 0 16px 0; font-size: 20px; color: #212529; display: flex; align-items: center; gap: 10px;"><span style="font-size: 22px;">📜</span> Related WCAG Success Criteria</h3>
<ul style="margin: 0; padding-left: 24px; color: #212529;">
<li style="margin-bottom: 8px;"><strong>3.3.2 Labels or Instructions (Level A):</strong> Labels or instructions are provided when content requires user input</li>
<li style="margin-bottom: 8px;"><strong>3.3.1 Error Identification (Level A):</strong> If an input error is automatically detected, the item that is in error is identified and the error is described in text</li>
<li style="margin-bottom: 8px;"><strong>3.3.3 Error Suggestion (Level AA):</strong> If an input error is automatically detected and suggestions are known, they are provided to the user</li>
<li><strong>1.3.5 Identify Input Purpose (Level AA):</strong> The purpose of each input field collecting user information can be programmatically determined</li>
</ul>
</div>
<div style="background-color: #fff; border: 1px solid #dee2e6; border-radius: 8px; padding: 24px; margin-bottom: 32px; border-left: 5px solid #f59f00;">
<h3 style="margin: 0 0 20px 0; font-size: 20px; color: #212529; display: flex; align-items: center; gap: 10px;"><span style="font-size: 22px;">💡</span> Key Concepts</h3>
<div style="margin-bottom: 28px;">
<h4 style="margin: 0 0 12px 0; font-size: 17px; color: #212529; padding-bottom: 8px; border-bottom: 2px solid #fff3bf; display: flex; align-items: center; gap: 10px;">Labels Are Accessible Names <span style="background-color: #d0ebff; color: #1864ab; padding: 2px 8px; border-radius: 4px; font-size: 11px; font-family: monospace;">1D.01</span></h4>
<p style="margin: 0 0 16px 0; color: #212529;">Form labels serve as the accessible names for form controls. Every form field needs:</p>
<ul style="margin: 0 0 16px 0; padding-left: 24px; color: #212529;">
<li style="margin-bottom: 6px;"><strong>Visible:</strong> Users can see what the field is for</li>
<li style="margin-bottom: 6px;"><strong>Persistent:</strong> The label stays visible after entering data</li>
<li style="margin-bottom: 6px;"><strong>Associated:</strong> The label is programmatically connected to the field</li>
<li><strong>Descriptive:</strong> The label clearly describes the expected input</li>
</ul>
<div style="background-color: #ffe3e3; border-radius: 6px; padding: 14px 16px; border-left: 4px solid #fa5252;">
<p style="margin: 0; font-size: 14px; color: #c92a2a;"><strong>Common failure:</strong> Using placeholder text as the only label. Placeholders disappear when users start typing, leaving them with no way to know what information was requested.</p>
</div>
</div>
<div style="margin-bottom: 28px;">
<h4 style="margin: 0 0 12px 0; font-size: 17px; color: #212529; padding-bottom: 8px; border-bottom: 2px solid #fff3bf; display: flex; align-items: center; gap: 10px;">Error Messages That Help <span style="background-color: #d0ebff; color: #1864ab; padding: 2px 8px; border-radius: 4px; font-size: 11px; font-family: monospace;">1D.02</span></h4>
<p style="margin: 0 0 16px 0; color: #212529;">WCAG requires error messages to do two things:</p>

<div style="display: flex; flex-wrap: wrap; gap: 16px; margin-bottom: 16px;">
<div style="flex: 1; min-width: 250px; background-color: #d3f9d8; border-radius: 6px; padding: 16px;">
<h5 style="margin: 0 0 10px 0; font-size: 14px; color: #1e7b34;">✓ Good error message</h5>
<p style="margin: 0; font-size: 14px; color: #212529;">"Please enter a valid email address, for example: name@company.com"</p>
<p style="margin: 8px 0 0 0; font-size: 13px; color: #495057;">Identifies problem AND suggests fix</p>
</div>
<div style="flex: 1; min-width: 250px; background-color: #ffe3e3; border-radius: 6px; padding: 16px;">
<h5 style="margin: 0 0 10px 0; font-size: 14px; color: #a61e1e;">✗ Unhelpful error message</h5>
<p style="margin: 0; font-size: 14px; color: #212529;">"Invalid input"</p>
<p style="margin: 8px 0 0 0; font-size: 13px; color: #495057;">Vague, no guidance for fixing</p>
</div>
</div>
<div style="background-color: #fff3bf; border-radius: 6px; padding: 14px 16px; border-left: 4px solid #f59f00;">
<p style="margin: 0; font-size: 14px; color: #212529;"><strong>Remember:</strong> Color alone cannot indicate errors (SC 1.4.1). Always include text descriptions and consider using icons as additional cues.</p>
</div>
</div>
<div>
<h4 style="margin: 0 0 12px 0; font-size: 17px; color: #212529; padding-bottom: 8px; border-bottom: 2px solid #fff3bf; display: flex; align-items: center; gap: 10px;">Input Purpose and Autocomplete <span style="background-color: #d0ebff; color: #1864ab; padding: 2px 8px; border-radius: 4px; font-size: 11px; font-family: monospace;">1D.03</span></h4>
<p style="margin: 0 0 16px 0; color: #212529;">SC 1.3.5 requires that the purpose of input fields collecting personal information can be programmatically determined. This enables autocomplete, which:</p>
<ul style="margin: 0 0 16px 0; padding-left: 24px; color: #212529;">
<li style="margin-bottom: 6px;"><strong>Motor disabilities:</strong> Reduces keystrokes, helping users who have difficulty typing</li>
<li style="margin-bottom: 6px;"><strong>Cognitive disabilities:</strong> Reduces memory demands, helping users who struggle to recall information</li>
<li><strong>All users:</strong> Faster form completion with fewer errors</li>
</ul>
<div style="background-color: #d0ebff; border-radius: 6px; padding: 14px 16px; border-left: 4px solid #1971c2;">
<p style="margin: 0; font-size: 14px; color: #1864ab;"><strong>Designer's role:</strong> Understand why autocomplete matters and which fields benefit from it. The technical implementation (autocomplete attributes) is an engineering responsibility.</p>
</div>
