# Accessible Names Learning Objectives
## Design Accessibility Certified Expert - Progressive Badge System

Based on accessibility standards (UC-WCAG, WCAG, Section 508, EN 301 549) and Adobe-specific requirements, these learning objectives guide the development of accessible names training for Adobe Designers across web, mobile, desktop, and document platforms.

This set of requirements covers images, icons, and decorative content. Either provide an accessible name for that element or instruct Engineering to mark it decorative so it can stay out of the way of the Accessibility API. 

---

## **DISCOVER BADGE - Testing Foundation**
*Prerequisites: None*

### **Knowledge Objectives**
- **Define accessible names** and their role in connecting humans to digital interfaces through assistive technology
- **Identify when accessible names are required** (always, sometimes, never) for different UI elements across platforms
- **Understand accessibility compliance requirements** and Adobe's legal obligations across platforms
- **Recognize common accessible name failures** and their impact on users

### **Skills Objectives**
- **Test designs against the Adobe Design Accessibility Checklist** accessible name section
- **Use platform-appropriate testing tools** to identify elements requiring accessible names
- **Recognize when UI components require accessible names** across platforms and when they don't
- **Identify generic interactive text** that needs expansion for accessibility

### **Application Objectives**
- **Consistently flag accessible name issues** during design reviews
- **Use Adobe's annotation systems** to mark elements requiring attention
- **Communicate basic accessible name requirements** to team members

### **Assessment Methods**
- **Checklist-based testing** of sample interfaces (90%+ accuracy required)
- **Identification exercises** with various UI elements$$
- **Basic Stark tool usage** demonstration

---

## **DEFINE BADGE - Assistive Technology Expertise**
*Prerequisites: Discover Accessibility Badge*

### **Knowledge Objectives**
- **Understand the naming method hierarchy** and when to use each approach (visible text, programmatic labels, platform-specific attributes, fallbacks)
- **Know accessible name calculation precedence** and how different platforms determine names
- **Understand quality guidelines** for writing effective accessible names
- **Recognize when graphics are decorative** and should be ignored by assistive technology
- **Master assistive technology usage** across platforms (screen readers, voice control, switch navigation)

### **Skills Objectives**
- **Create design annotations** using appropriate tools to specify accessible names for engineers
- **Write clear, concise accessible names** that convey function over form
- **Use assistive technology testing** to validate accessible name implementation across platforms
- **Determine when generic CTAs need expansion** and collaborate with content designers
- **Apply accessible name concepts** to Adobe's design systems across platforms

### **Application Objectives**
- **Consistently annotate** all interactive elements requiring accessible names during design handoffs
- **Proactively identify** generic link text that needs expansion during content review
- **Prioritize visible text** solutions over programmatic labels when possible across all platforms
- **Integrate accessible name considerations** into the design workflow at the appropriate stage

### **Assessment Methods**
- **Implementation projects** creating blueline annotations for complex interfaces
- **Screen reader validation** of accessible name solutions
- **Collaboration exercises** with content designers on link text expansion
- **Quality evaluation** of accessible name writing

---

## **DELIVER BADGE - Independent Review**
*Prerequisites: Define Accessibility Badge*

### **Knowledge Objectives**
- **Master accessibility API relationships** between design specifications and assistive technology users
- **Understand complex edge cases** and ambiguous accessible name scenarios
- **Know advanced quality guidelines** for challenging UI patterns
- **Understand mentoring and training approaches** for accessibility
- **Lead accessibility reviews** and provide expert guidance

### **Skills Objectives**
- **Handle complex accessible name scenarios** that don't fit standard patterns
- **Mentor Discover and Define Badge designers** on accessible name requirements
- **Conduct independent design accessibility reviews** for other designers' work
- **Advise on accessible name requirements** within Adobe's ecosystem across all platforms
- **Make judgment calls** on ambiguous accessible name situations

### **Application Objectives**
- **Lead accessibility reviews** and provide expert guidance to design teams
- **Train others** on accessible name best practices
- **Handle complex cross-functional collaboration** on challenging accessible name scenarios
- **Contribute to Adobe's accessibility standards** and process improvements

### **Assessment Methods**
- **Complex scenario reviews** with ambiguous accessible name situations
- **Mentoring demonstrations** with Discover/Define Badge designers
- **Independent project leadership** on accessible name implementation
- **Expert consultation** on challenging accessibility questions

---

## **Progressive Learning Outcomes**

### **Discover Accessibility Badge Completion**
- **100% of participants** can identify elements requiring accessible names across web, mobile, desktop, and document interfaces
- **Participants demonstrate** basic accessibility testing tool usage across relevant platforms
- **Participants explain** the connection between accessible names and assistive technology users

### **Define Accessibility Badge Completion**
- **100% of participants** can create proper design annotations for all platforms
- **Participants write** quality accessible names following established guidelines
- **Participants validate** solutions using appropriate assistive technology testing for each platform
- **Reduction in accessible name-related defects** in audited designs from trained participants

### **Deliver Accessibility Badge Completion**
- **Participants conduct** independent design accessibility reviews across all platforms
- **Participants mentor** other designers on accessible name requirements
- **Participants handle** complex, ambiguous accessible name scenarios
- **Measurable improvement** in Adobe's accessibility audit scores for products touched by trained designers

---

*This progressive structure ensures each badge level builds upon previous knowledge while developing increasingly sophisticated skills and responsibilities in accessible name implementation across web, mobile, desktop, and document platforms.*
