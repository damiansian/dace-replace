Week 4 Applied Practice: Navigation Accessibility Audit

Landmarks, Navigation Consistency & Skip Links Project

🔍 Discover Badge 🏷️ Module 1G 📝 16 Points

This applied practice project brings together the navigation skills you learned in Week 4: landmark regions, consistent navigation, and skip links. These skills ensure users can navigate efficiently with screen readers and keyboards, jumping directly to major page sections and bypassing repeated content.

You will receive a multi-page design mockup and create a complete navigation accessibility audit with Blueline annotations. This mirrors real-world design handoff where navigation structure specifications are delivered to engineering.

🎯 What You'll Demonstrate

Landmarks & Structure

  • Identify all landmark regions (banner, navigation, main, complementary, contentinfo, search)
  • Assign unique accessible names when multiple landmarks of the same type exist
  • Document landmark structure using Stark Blueline annotations
  • Note HTML5 semantic equivalents for each landmark

Navigation Consistency

  • Audit navigation consistency across all pages
  • Document the order of navigation items on each page
  • Flag any inconsistencies where order changes between pages
  • Provide specific recommendations for fixing issues

Skip Links & Bypass Blocks

  • Identify repeated content blocks that users need to bypass
  • Specify skip link placement (first focusable element)
  • Document skip link destination targets
  • Specify visibility behavior (hidden until focus or always visible)

📋 Learning Outcomes Covered

ID Outcome

1G.02 Design consistent navigation by creating patterns that appear in the same relative order across pages and states

📦 Assignment Instructions

What You Receive

A multi-page design mockup in Figma containing navigation structures requiring accessibility specifications.

📎 Duplicate Figma File to Begin

The mockup includes:

  • Three linked pages (Home, Products, About) with consistent header/footer
  • Primary and secondary navigation components
  • Sidebar with complementary content
  • Search functionality in the header
  • Footer with additional navigation links

What You Deliver

An annotated Figma file with Stark Blueline annotations covering:

Part 1: Landmark Annotations

  • All landmark regions identified with correct ARIA roles
  • Unique accessible names for duplicate landmarks (e.g., "Primary navigation" vs "Footer navigation")
  • HTML5 semantic equivalent noted (e.g., banner = <header>)

Part 2: Navigation Consistency Audit

  • Cross-page navigation comparison documenting order on each page
  • Consistency issues flagged with specific locations
  • Recommendations for fixing any inconsistencies

Part 3: Skip Link Specifications

  • Skip link placement specified (first focusable element)
  • Destination target documented (e.g., #main-content)
  • Visibility behavior specified (hidden until focus or always visible)

Step-by-Step Process

  1. Map landmark regions: Review each page and identify all landmark regions. Annotate with role names and accessible names where needed.
  2. Audit navigation consistency: Compare navigation across all three pages. Document the order of items on each page and flag any inconsistencies.
  3. Specify skip links: Identify repeated content blocks (header, navigation). Specify skip link placement as first focusable element, destination target, and visibility behavior.
  4. Final review: Verify all annotations are complete and engineering can implement without follow-up questions.

💡 Tips for Success

Landmarks

  • Six landmark types: banner, navigation, main, complementary, contentinfo, search
  • One main per page: Every page should have exactly one main landmark
  • Name duplicates: Two nav elements need labels like "Primary navigation" and "Footer navigation"
  • Context matters: A <header> inside an <article> is NOT a banner landmark

Navigation Consistency

  • Same order everywhere: If nav is Home | Products | About on page 1, it must be the same on all pages
  • Document the order: Write out the exact sequence for each page to compare
  • Check mobile vs desktop: Navigation order should be consistent across breakpoints
  • Reference SC 3.2.3: Consistent Navigation requires same relative order

Skip Links

  • First focusable element: Skip link must be the very first thing a keyboard user can Tab to
  • Document destination: "Skip to main content" must specify where focus moves (e.g., #main-content)
  • Visibility options: Hidden until focus (appears on Tab), or always visible (specify which)
  • Three bypass methods: Skip links, landmarks, and headings all satisfy SC 2.4.1
