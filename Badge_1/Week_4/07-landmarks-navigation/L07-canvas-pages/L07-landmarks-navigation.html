Lesson 07: Landmarks & Navigation

🔍 Discover Badge 🧭 Module 1G

Landmarks, skip links, and consistent navigation give users multiple ways to move through your designs efficiently. Screen reader users rely on landmarks to jump to major sections. Keyboard users depend on skip links to bypass repeated content. Everyone benefits from predictable navigation patterns.

In this lesson, you'll learn how to specify landmark regions, design consistent navigation, implement skip links, and ensure focus order supports keyboard accessibility.

💡 Key Concepts

Landmark Regions 1G.01

Landmarks are structural markers that identify major page sections. Screen readers let users jump directly to these regions, similar to how chapters work in a book.

ARIA Role HTML5 Element Purpose
banner <header> Site header, logo, site-wide info
navigation <nav> Navigation links
main <main> Primary page content
complementary <aside> Supporting content, sidebars
contentinfo <footer> Site footer, copyright
search <search> or role="search" Search functionality

When you have multiple landmarks of the same type: Each must have a unique accessible name. For example, two nav elements need labels like "Primary navigation" and "Footer navigation".

Consistent Navigation 1G.02

Navigation components that appear on multiple pages must maintain the same relative order. Users build mental models of site structure, and inconsistency forces them to relearn navigation on each page.

✓ Consistent (Correct)

Page 1: Home | Products | About | Contact
Page 2: Home | Products | About | Contact

✗ Inconsistent (Failure)

Page 1: Home | Products | About | Contact
Page 2: Home | About | Products | Contact

Skip Links & Bypass Blocks 1G.03

SC 2.4.1 requires mechanisms to bypass repeated content. Keyboard users shouldn't have to Tab through 50 navigation links on every page. Three methods satisfy this requirement:

  1. Skip links: "Skip to main content" links at page start
  2. Landmarks: Enable jumping to page regions
  3. Headings: Allow navigation via heading hierarchy

Skip link placement: Must be the first focusable element on the page. When activated, focus moves to the target region (often main content).

Focus Order 1G.04

SC 2.4.3 requires focus order to preserve meaning and operability. For left-to-right languages, focus typically follows left-to-right, top-to-bottom visual order.

Dynamic content requires focus management:

  • Modal dialogs: Focus moves in, traps within, returns to trigger on close
  • Accordions: Focus stays on trigger, expanded content becomes available
  • Tab panels: Arrow keys navigate within, Tab moves out
