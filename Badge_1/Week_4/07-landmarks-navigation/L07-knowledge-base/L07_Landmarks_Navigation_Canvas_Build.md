<h2 style="margin: 0 0 12px 0; font-size: 28px; color: #212529;">Lesson 07: Landmarks &amp; Navigation</h2>
<div style="display: flex; flex-wrap: wrap; gap: 14px; margin-top: 18px;"><span style="display: inline-flex; align-items: center; gap: 8px; background-color: #e6fcf5; color: #087f5b; padding: 10px 18px; border-radius: 24px; font-size: 15px;"> <span style="font-size: 18px;">🔍</span> Discover Badge </span> <span style="display: inline-flex; align-items: center; gap: 8px; background-color: #e7f5ff; color: #1864ab; padding: 10px 18px; border-radius: 24px; font-size: 15px;"> <span style="font-size: 18px;">🧭</span> Module 1G </span></div>
</div>
<p style="font-size: 17px; color: #495057; margin-bottom: 16px; padding: 0 4px;">Landmarks, skip links, and consistent navigation give users multiple ways to move through your designs efficiently. Screen reader users rely on landmarks to jump to major sections. Keyboard users depend on skip links to bypass repeated content. Everyone benefits from predictable navigation patterns.</p>
<p style="font-size: 17px; color: #495057; margin-bottom: 32px; padding: 0 4px;">In this lesson, you'll learn how to specify landmark regions, design consistent navigation, implement skip links, and ensure focus order supports keyboard accessibility.</p>

<h4 style="margin: 0 0 12px 0; font-size: 17px; color: #212529; padding-bottom: 8px; border-bottom: 2px solid #d0ebff;">Knowledge Objectives</h4>
<ul style="margin: 0 0 28px 0; padding-left: 24px; color: #212529; font-size: 14px;">
<li style="margin-bottom: 10px;">Define the six ARIA landmark roles (banner, navigation, main, complementary, contentinfo, search) and their HTML5 semantic equivalents.</li>
<li style="margin-bottom: 10px;">Explain when multiple landmarks of the same type need unique accessible names.</li>
<li style="margin-bottom: 10px;">Describe WCAG 3.2.3 Consistent Navigation requirements and why navigation order matters.</li>
<li style="margin-bottom: 10px;">Identify the three methods to satisfy SC 2.4.1 Bypass Blocks: skip links, landmarks, and headings.</li>
<li style="margin-bottom: 10px;">Explain how focus order relates to visual layout and why it must preserve meaning and operability.</li>
<li>Recognize how dynamic content (modals, accordions) affects focus order and requires focus management.</li>

<h3 style="margin: 0 0 20px 0; font-size: 20px; color: #212529; display: flex; align-items: center; gap: 10px;"><span style="font-size: 22px;">💡</span> Key Concepts</h3>
<div style="margin-bottom: 28px;">
<h4 style="margin: 0 0 12px 0; font-size: 17px; color: #212529; padding-bottom: 8px; border-bottom: 2px solid #fff3bf; display: flex; align-items: center; gap: 10px;">Landmark Regions <span style="background-color: #d0ebff; color: #1864ab; padding: 2px 8px; border-radius: 4px; font-size: 11px; font-family: monospace;">1G.01</span></h4>
<p style="margin: 0 0 16px 0; color: #212529;">Landmarks are structural markers that identify major page sections. Screen readers let users jump directly to these regions, similar to how chapters work in a book.</p>

<p style="margin: 0; font-size: 14px; color: #1864ab;"><strong>When you have multiple landmarks of the same type:</strong> Each must have a unique accessible name. For example, two nav elements need labels like "Primary navigation" and "Footer navigation".</p>

<div style="margin-bottom: 28px;">
<h4 style="margin: 0 0 12px 0; font-size: 17px; color: #212529; padding-bottom: 8px; border-bottom: 2px solid #fff3bf; display: flex; align-items: center; gap: 10px;">Consistent Navigation <span style="background-color: #d0ebff; color: #1864ab; padding: 2px 8px; border-radius: 4px; font-size: 11px; font-family: monospace;">1G.02</span></h4>
<p style="margin: 0 0 16px 0; color: #212529;">Navigation components that appear on multiple pages must maintain the same relative order. Users build mental models of site structure, and inconsistency forces them to relearn navigation on each page.</p>
<div style="display: flex; flex-wrap: wrap; gap: 16px; margin-bottom: 16px;">

<div style="margin-bottom: 28px;">
<h4 style="margin: 0 0 12px 0; font-size: 17px; color: #212529; padding-bottom: 8px; border-bottom: 2px solid #fff3bf; display: flex; align-items: center; gap: 10px;">Skip Links &amp; Bypass Blocks <span style="background-color: #d0ebff; color: #1864ab; padding: 2px 8px; border-radius: 4px; font-size: 11px; font-family: monospace;">1G.03</span></h4>
<p style="margin: 0 0 16px 0; color: #212529;">SC 2.4.1 requires mechanisms to bypass repeated content. Keyboard users shouldn't have to Tab through 50 navigation links on every page. Three methods satisfy this requirement:</p>
<ol style="margin: 0 0 16px 0; padding-left: 24px; color: #212529;">
<li style="margin-bottom: 8px;"><strong>Skip links:</strong> "Skip to main content" links at page start</li>
<li style="margin-bottom: 8px;"><strong>Landmarks:</strong> Enable jumping to page regions</li>
<li><strong>Headings:</strong> Allow navigation via heading hierarchy</li>
</ol>
<div style="background-color: #fff3bf; border-radius: 6px; padding: 14px 16px; border-left: 4px solid #f59f00;">
<p style="margin: 0; font-size: 14px; color: #212529;"><strong>Skip link placement:</strong> Must be the first focusable element on the page. When activated, focus moves to the target region (often main content).</p>
</div>
</div>
<div>
<h4 style="margin: 0 0 12px 0; font-size: 17px; color: #212529; padding-bottom: 8px; border-bottom: 2px solid #fff3bf; display: flex; align-items: center; gap: 10px;">Focus Order <span style="background-color: #d0ebff; color: #1864ab; padding: 2px 8px; border-radius: 4px; font-size: 11px; font-family: monospace;">1G.04</span></h4>
<p style="margin: 0 0 16px 0; color: #212529;">SC 2.4.3 requires focus order to preserve meaning and operability. For left-to-right languages, focus typically follows left-to-right, top-to-bottom visual order.</p>
<p style="margin: 0 0 8px 0; color: #212529;"><strong>Dynamic content requires focus management:</strong></p>
<ul style="margin: 0; padding-left: 24px; color: #212529;">
<li style="margin-bottom: 6px;"><strong>Modal dialogs:</strong> Focus moves in, traps within, returns to trigger on close</li>
<li style="margin-bottom: 6px;"><strong>Accordions:</strong> Focus stays on trigger, expanded content becomes available</li>
<li><strong>Tab panels:</strong> Arrow keys navigate within, Tab moves out</li>
</ul>
</div>
