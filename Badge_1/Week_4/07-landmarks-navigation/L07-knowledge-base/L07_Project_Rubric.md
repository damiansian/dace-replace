# L07: Landmarks & Navigation - Project Rubric

**Course:** DACE (Design Accessibility Certified Expert)  
**Badge:** Discover  
**Lesson:** L07  
**Assessment Type:** Applied Skill Project  
**Project Name:** Navigation Accessibility Audit  
**Points:** 15  
**Format:** Rubric-graded by instructor  
**Outcomes Tested:** 1G.01, 1G.02, 1G.03, 1G.04

---

## Project Overview

Students receive a multi-page design mockup with navigation accessibility issues. They conduct a complete navigation audit covering landmarks, consistent navigation, skip links, and focus order, then document their findings with blueline annotations.

### Project Deliverables

1. **Landmark Region Analysis**
   - Identification of all landmark regions across pages
   - Appropriate landmark role assignments
   - Unique accessible names for multiple landmarks of the same type
   - Landmark annotation using Stark

2. **Navigation Consistency Audit**
   - Cross-page comparison of navigation components
   - Documentation of consistency issues
   - Verification of navigation order across pages
   - Recommendations for improving consistency

3. **Skip Link Specifications**
   - Identification of repeated content blocks
   - Skip link placement recommendations
   - Skip link destination specifications
   - Visibility and focus behavior notes

4. **Focus Order Verification**
   - Keyboard testing of focus order
   - Focus order documentation for complex layouts
   - Dynamic content focus management specifications
   - Focus order annotations

5. **Annotated Figma File**
   - Stark Blueline annotations for all findings
   - Landmark specifications with roles and names
   - Navigation consistency notes
   - Skip link and focus order documentation
   - WCAG SC references

## Grading Guidelines for Instructors

### Scoring Approach

1. **Evaluate each criterion independently** against the descriptors
2. **Use the point ranges provided** (e.g., 3.4-3.75 for Exceeds)
3. **Provide written feedback** explaining the score for each criterion
4. **Identify patterns** to help students improve

### Feedback Template

For each criterion, provide:
- **Strengths:** What the student did well
- **Areas for improvement:** Specific gaps or errors
- **Resources:** Links to WCAG docs or examples for improvement

### Example Feedback

**Criterion 1: Landmark Region Identification (3.5/3.75 - Exceeds)**

*Strengths:* You identified all landmark regions across all three pages and correctly assigned roles. Your naming convention for the two navigation landmarks ("Primary navigation" and "Footer links") is clear and follows best practices.

*Areas for improvement:* Consider noting that the search form could be wrapped in a search landmark for better discoverability.

*Overall:* Excellent landmark analysis demonstrating mastery beyond requirements.

## Common Student Mistakes to Watch For

### Landmark Region Identification

- Not identifying all six potential landmark types
- Using generic names instead of descriptive names for duplicate landmarks
- Missing landmarks in footer or complementary content
- Not recognizing that `<header>` inside `<article>` is not a banner landmark
- Confusing section elements with landmarks

### Navigation Consistency Analysis

- Only auditing one page instead of comparing across pages
- Not documenting the specific order of navigation items
- Missing mobile vs desktop consistency checks
- Overlooking logged-in vs logged-out navigation differences
- Not referencing SC 3.2.3 requirements

### Skip Link Specifications

- Not specifying skip link placement (must be first focusable)
- Missing skip link destination targets
- Not documenting visibility behavior
- Forgetting focus must actually move when skip link is activated
- Assuming landmarks alone satisfy bypass blocks

### Focus Order Verification

- Not testing with keyboard only
- Missing complex layout analysis
- Not documenting dynamic content focus management
- Forgetting modal focus trap and return
- Confusing visual order with DOM order

## Instructor Calibration Notes

### Consistency Across Graders

If multiple instructors grade:
- Review sample submissions together
- Calibrate on what constitutes "Exceeds" vs "Meets" vs "Near"
- Document edge cases and how to score them
- Share feedback examples

### Edge Cases

**Q: Student identifies all landmarks but uses informal names like "top nav" instead of "Primary navigation." Deduct points?**  
A: Minor deduction in "Meets" range. Names should be professional and descriptive, but understanding is correct.

**Q: Student audits navigation but only compares two of four pages. How to score?**  
A: "Near Mastery" range. Concept is understood but execution is incomplete.

**Q: Student specifies skip links but doesn't document where focus should move. Score?**  
A: "Near Mastery" range. Skip link destination is a key requirement.

**Q: Student goes beyond requirements (tests with screen reader, provides code examples). Score?**  
A: "Exceeds Mastery" and note as exemplary work.
