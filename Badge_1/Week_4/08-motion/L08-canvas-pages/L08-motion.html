Lesson 08: Motion & Animation

🔍 Discover Badge 🎬 Module 1H

Motion and animation can enhance user experience, but they can also cause real harm. Users with vestibular disorders experience dizziness and nausea from parallax effects and animated transitions. Flashing content can trigger seizures. Auto-playing videos and carousels distract users with attention-related disabilities.

In this lesson, you'll learn to design motion that respects user preferences, implement pause controls for auto-playing content, avoid seizure-inducing patterns, and create reduced-motion alternatives that preserve meaning.

💡 Key Concepts

Why Motion Accessibility Matters ALL

Motion on screen can cause real physical harm. Unlike many accessibility issues that create inconvenience, motion problems can trigger medical conditions.

Who is affected:

  • Vestibular disorders: Parallax scrolling and animated transitions cause dizziness, nausea, and disorientation
  • Seizure disorders: Flashing content can trigger photosensitive epileptic seizures
  • Attention disabilities: Auto-playing videos and carousels distract from primary content
  • Cognitive disabilities: Complex animations increase cognitive load and confusion

Critical: Motion-triggered seizures can be life-threatening. This is not about user preference. It is about safety.

The 5-Second Rule 1H.02 SC 2.2.2

Key rule: Any automatic movement that lasts more than 5 seconds must have a mechanism to pause, stop, or hide it.

SC 2.2.2 (Pause, Stop, Hide) applies to content that:

  • Starts automatically (not triggered by user action)
  • Lasts more than 5 seconds
  • Appears alongside other content (not the only thing on screen)

Common examples requiring pause controls:

  • Auto-playing video backgrounds
  • Carousels and slideshows
  • Animated illustrations or mascots
  • Parallax scrolling effects
  • Loading animations (if longer than 5 seconds)
✓ Compliant
  • Visible pause/play button
  • Animation stops within 5 seconds
  • User-initiated animation only
✗ Failure
  • Infinite looping with no controls
  • Pause only works for current page
  • Controls hidden or hard to find

Reduced Motion Preferences 1H.01 SC 2.3.3

Users can set a "reduce motion" preference in their operating system settings. Designs must honor this preference using the prefers-reduced-motion media query.

@media (prefers-reduced-motion: reduce) {
  /* Disable or reduce animations */
  .animated-element { animation: none; }
}

When reduced motion is enabled:

  • Replace animations with instant state changes
  • Disable parallax scrolling effects
  • Stop auto-playing videos (or show static poster)
  • Use simple fades instead of complex transitions

Designer's role: Document in your specs how each animation should behave when prefers-reduced-motion is enabled. Do not leave this for engineering to guess.

Seizure Risk and Flash Thresholds 1H.02 SC 2.3.1

Hard limit: Content must not flash more than 3 times per second. This threshold exists because flashing can trigger seizures in people with photosensitive epilepsy.

What counts as a "flash":

  • Rapid alternation between light and dark
  • Strobe effects or blinking elements
  • Video content with rapid scene changes
  • Animated GIFs with quick frame transitions

Area matters:

The flash threshold applies when the flashing area is large enough: more than 25% of 10 degrees of visual field (roughly 341x256 pixels at typical viewing distance). Small, isolated flashing elements may be acceptable, but always err on the side of caution.

Red flash is especially dangerous

Saturated red flashing is particularly likely to trigger seizures. Avoid red strobing effects entirely.

Designing Motion Alternatives 1H.03

When animation communicates meaning, you need a reduced-motion alternative that preserves that meaning without the motion.

Essential vs. decorative motion:

  • Essential: Progress indicators, state changes, drag-and-drop feedback
  • Decorative: Hover effects, page transitions, background animations
