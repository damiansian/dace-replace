### Frame 1.2: What makes feedback useful

**Type:** Four-grid

**Headline:** Four traits of constructive feedback

**Cards:**
- **Specific:** Name the element, state, or flow
- **Actionable:** Make the next step clear
- **User-focused:** Explain the effect on users
- **Kind:** Keep critique about the work, not the person

**Key insight callout:**
If the team cannot tell what to do next, the feedback is incomplete.

**Speaker notes:**
Constructive feedback is not softer feedback. It is clearer feedback. Precision and kindness are both part of quality.

---

### Frame 1.3: SBI and COIN

**Type:** Two-column

**Headline:** Two useful feedback frameworks

**Left column:**
- Heading: `SBI`
- Situation
- Behavior
- Impact
- Best when the issue is concrete and localized

**Right column:**
- Heading: `COIN`
- Context
- Observation
- Impact
- Next
- Best when the team needs a clearer path forward

**Speaker notes:**
Neither framework is mandatory. They are tools that help make critique more disciplined. Use them when they improve clarity.

### Frame 2.2: Structure of a short accessibility critique

**Type:** Process-flow

**Headline:** A simple critique sequence

**Content:**
- Start with scope and goals
- Review findings in priority order
- Keep discussion anchored in user impact
- End with clear next steps

**Speaker notes:**
Facilitation matters because good findings can still get lost in a poorly run discussion.

---

### Frame 2.3: Psychological safety and de-escalation

**Type:** Two-column

**Headline:** Keep the critique about the work

**Left column:**
- Heading: `What helps`
- Precise language
- Questions that clarify
- Re-anchoring on users
- Clear distinction between person and artifact

**Right column:**
- Heading: `What makes it worse`
- Loaded language
- Public correction without context
- Debating intent instead of user impact
- Letting side arguments take over

**Key insight callout:**
Defensiveness often drops when the conversation returns to the specific barrier and the affected user.

**Speaker notes:**
Psychological safety is not the same as lowering the standard. It is what allows the standard to be discussed productively.

### Frame 3.2: Adjusting for skill level

**Type:** Three-column

**Headline:** The same issue, different explanation

**Content:**
- Discover Badge designer: explain the principle and the user impact
- Define Badge designer: connect the issue to AT behavior and validation
- Untrained designer: keep it concrete, avoid jargon, explain the consequence

**Key insight callout:**
Good critique is calibrated. The same wording does not work equally well for every audience.

**Speaker notes:**
This is not about lowering rigor. It is about matching the explanation to the listener's current understanding.

---

### Frame 3.3: Checkpoint - Audience adaptation

**Type:** Checklist

**Headline:** Checkpoint: Audience adaptation

**Checklist:**
- [ ] I can adapt a critique for different skill levels
- [ ] I understand the difference between a mentoring moment and a review moment
- [ ] I know how to keep the recommendation clear even when the explanation changes

**Speaker notes:**
This closes the communication loop. Good critique depends on both technical accuracy and audience calibration.
