Lesson 10: Screen Reader Fundamentals

🎯 Define Badge 🏷️ Module 2B

Screen readers are the primary assistive technology for people who are blind or have low vision. Understanding how screen readers work transforms how designers think about accessibility, because you hear your design the way AT users experience it.

In this lesson, you'll learn how to navigate web content using VoiceOver (desktop and iOS), NVDA, and TalkBack. You'll understand the critical distinction between browse mode and focus mode on desktop, learn that mobile screen readers are essentially browse mode only, and practice efficient navigation using quick keys, the Rotor, and Reading Controls. This builds directly on L09's accessibility tree foundation: now you'll hear what you've been seeing.

💡 Key Concepts

How Screen Readers Work

Screen readers are software that reads the accessibility tree aloud (not the visual display). They convert the name, role, state, and value properties you learned about in L09 into speech output. When a screen reader encounters a button, it announces something like "Submit, button" (the name followed by the role).

The Connection to L09

Accessibility Tree (what you inspected) → Screen Reader (what users hear). Same data, different output. If the tree is wrong, the announcements will be wrong.

Browse Mode vs Focus Mode

Screen readers have two navigation modes that serve different purposes:

Browse Mode (Reading Mode)

Navigate through all content: headings, text, images, links. Arrow keys move between elements. Single-key shortcuts (H for headings, D for landmarks) are active. Used for reading and scanning page content.

Focus Mode (Forms Mode)

Interact with form fields and widgets. Arrow keys control the focused element (text cursor, dropdown options). Single-key shortcuts are disabled so keystrokes go to the control. Used for filling forms and operating interactive components.

Mobile: Browse Mode Only

iOS VoiceOver and TalkBack do not have a separate focus mode. All navigation uses gestures: swipe right/left to move between elements, double-tap to activate. When you reach a text field, the on-screen keyboard appears automatically. The Rotor (iOS) and Reading Controls (TalkBack) provide quick navigation without mode switching.

Designer Impact: On desktop, incorrect ARIA roles can force screen readers into the wrong mode. On mobile, the same semantic structure is needed for Rotor and Reading Controls to work. Proper headings, landmarks, and labels are required across all platforms.

Quick Navigation

Real screen reader users rarely read pages from top to bottom. According to WebAIM surveys, 67.5% of screen reader users navigate by headings as their primary strategy. Quick navigation works across all platforms:

Action NVDA (Desktop) VoiceOver (Desktop) iOS / TalkBack (Mobile)
Next heading H Web Rotor (VO+U) Rotor / Reading Controls → Headings → Swipe
Heading by level 1-6 Web Rotor filtered Not available on mobile
Next landmark D Web Rotor → Landmarks Rotor / Reading Controls → Landmarks → Swipe
Next link K Web Rotor → Links Rotor / Reading Controls → Links → Swipe
Next form field F Web Rotor → Form Controls Rotor / Reading Controls → Controls → Swipe

Why This Matters: Your heading structure (L02) and landmark annotations (L07) directly create the navigation structure screen reader users depend on across all platforms. Flat headings and missing landmarks break quick navigation on desktop and mobile alike.

Essential Commands Reference

Keep these handy when testing:

VoiceOver (Mac)

  • On/Off: Cmd+F5
  • Next: VO+Right Arrow
  • Activate: VO+Space
  • Web Rotor: VO+U
  • Stop speech: Ctrl

NVDA (Windows)

  • Start: Ctrl+Alt+N
  • Quit: Insert+Q
  • Navigate: Arrow keys
  • Elements List: Insert+F7
  • Stop speech: Ctrl

iOS VoiceOver

  • Next: Swipe right
  • Previous: Swipe left
  • Activate: Double-tap
  • Rotor: Two-finger twist
  • Stop speech: Two-finger tap

TalkBack (Android)

  • Next: Swipe right
  • Previous: Swipe left
  • Activate: Double-tap
  • Reading Controls: Swipe up-down
  • Stop speech: Two-finger tap
