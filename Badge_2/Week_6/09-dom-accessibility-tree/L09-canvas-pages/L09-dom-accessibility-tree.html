Lesson 09: DOM and Accessibility Tree

🎯 Define Badge 🏷️ Module 2A

Understanding how browsers expose content to assistive technology is fundamental to designing accessible experiences. The accessibility tree is the interface between your designs and how AT users experience them.

In this lesson, you'll learn how the browser DOM becomes the accessibility tree, how to inspect accessibility properties using browser dev tools, and how to predict the impact of your design decisions on the accessibility tree.

💡 Key Concepts

What is the Accessibility Tree?

The accessibility tree is a simplified representation of the DOM that browsers expose to assistive technologies. While the DOM contains everything needed to render the visual page, the accessibility tree contains only what's needed for AT users to understand and interact with content.

The Pipeline

HTML/CSS → DOM → Accessibility Tree → Platform Accessibility API → Assistive Technology

Accessibility Tree Properties

Each node in the accessibility tree has four key properties:

Name

The accessible name (from labels, aria-label, text content, etc.)

Role

What type of element it is (button, link, heading, etc.)

State

Current condition (checked, expanded, disabled, etc.)

Value

Current value for inputs, sliders, etc.

Inspecting the Accessibility Tree

Modern browser dev tools let you inspect accessibility properties directly:

  • Chrome: Elements panel → Accessibility tab
  • Firefox: Accessibility panel (dedicated)
  • Safari: Elements panel → Node tab → Accessibility

Pro Tip: Before you test with a screen reader, always check the accessibility tree first. If the tree is wrong, the screen reader experience will be wrong.
