# L12: 2C Screen Magnification – Learning Outcomes Rubric

**Course:** DACE (Design Accessibility Certified Expert)  
**Badge:** Define  
**Lesson:** L12 – Screen Magnification  
**Competency Code:** 2C (Screen Magnification)  
**Outcomes:** 2C.01, 2C.02, 2C.03  
**Scale:** 4, 3, 2, 1 (Exceeds / Meets / Near / Below Mastery)

---

## Mastery Scale

| Points | Level | Description |
|--------|-------|-------------|
| 4 | **Exceeds Mastery** | Demonstrates advanced understanding beyond requirements; thorough, professional-level work |
| 3 | **Meets Mastery** | Fully meets the learning outcome requirements; competent, complete work |

**Passing Threshold:** Meets Mastery (3) or higher on all outcomes.

---

## Outcome 2C.01: Understand screen magnification tools and user needs

**Outcome statement:** Explain how ZoomText, Windows Magnifier, macOS Zoom, and browser zoom work, how they differ from each other, and the user populations they serve.

| Level | Points | Performance indicators |
|-------|--------|------------------------|
| **Exceeds Mastery** | 4 | Explains all major tools (ZoomText, Windows Magnifier, macOS Zoom, browser zoom) with accurate distinctions; clearly describes pixel magnification vs browser reflow; identifies user populations (low vision, aging, temporary impairment) and explains why magnification testing requires tools beyond browser zoom alone; communicates in terms suitable for design and engineering teams |
| **Meets Mastery** | 3 | Correctly explains how the main magnification tools work and how they differ; distinguishes system magnification from browser zoom; describes relevant user populations; can recommend appropriate tools for different platforms |
| **Near Mastery** | 2 | Partially correct explanation of tools; some confusion between system zoom and browser zoom or between user populations; shows understanding but gaps in clarity or completeness |
| **Below Mastery** | 1 | Incorrect or missing explanation of tools; does not distinguish system vs browser zoom; does not demonstrate understanding of magnification user needs or tool differences |

---

## Outcome 2C.02: Test designs with screen magnification tools

**Outcome statement:** Validate designs at various magnification levels using system and browser zoom, identifying issues that affect magnification users' ability to navigate and understand content.

| Level | Points | Performance indicators |
|-------|--------|------------------------|
| **Exceeds Mastery** | 4 | Tests systematically at 200%, 400%, and system zoom; documents findings with magnified viewport screenshots; identifies spatial separation, tooltip/popup positioning, focus visibility, and reflow issues; prioritizes by user impact; creates reusable checklist or notes for common patterns |
| **Meets Mastery** | 3 | Tests at multiple zoom levels (e.g., 200%, 400%); identifies major magnification issues (proximity, positioning, focus); documents findings clearly; shows competent use of magnification tools |
| **Near Mastery** | 2 | Conducts some magnification testing but misses key levels or issue types; documents partially; shows understanding of viewport concept but execution has gaps |
| **Below Mastery** | 1 | Minimal or no meaningful magnification testing; does not identify magnification-specific issues; does not demonstrate ability to test with system or browser zoom effectively |

---

## Outcome 2C.03: Design for screen magnification compatibility

**Outcome statement:** Create layouts and interaction patterns that work well for users who view content through a magnified viewport, maintaining spatial relationships and orientation cues.

| Level | Points | Performance indicators |
|-------|--------|------------------------|
| **Exceeds Mastery** | 4 | Evaluates layouts for proximity, reflow, and positioning; annotates designs with magnification-specific requirements; recommends layout changes that preserve spatial relationships at high zoom; specifies error message placement, tooltip behavior, and focus visibility; may produce design guidelines for tooltip/popup positioning |
| **Meets Mastery** | 3 | Correctly evaluates designs for magnification compatibility; identifies proximity and positioning issues; recommends appropriate improvements; annotates or specifies behavior that supports magnified viewport use |
| **Near Mastery** | 2 | Partially evaluates layouts; identifies some proximity or reflow issues but misses others; recommendations are vague or incomplete; shows understanding but execution has gaps |
| **Below Mastery** | 1 | Does not evaluate designs for magnification; misses critical proximity or positioning issues; recommendations missing or incorrect; does not demonstrate understanding of magnification-friendly design patterns |
