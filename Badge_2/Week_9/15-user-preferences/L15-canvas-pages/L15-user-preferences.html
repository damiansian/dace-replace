Lesson 15: User Preferences and System Settings

🎯 Define Badge 🏷️ Module 2F

Users configure accessibility preferences at the operating system level: reduced motion, dark mode, increased text size, and high contrast. Well-designed products detect and respond to these preferences automatically. As a designer, you need to know where these settings live, how to specify responsive behaviors in your annotations, and how to test that your designs adapt correctly.

In this lesson, you will learn the full preference landscape across platforms, how CSS media queries detect user preferences, and how to design and test for preference responsiveness. This lesson unifies concepts from L08 (reduced motion), L12 (text scaling), and L14 (high contrast) into a systematic approach to preference-responsive design.

💡 Key Concepts

CSS Media Queries for User Preferences

Browsers expose user preferences through CSS media queries. When a user enables a system-level accessibility setting, the browser makes the corresponding media query value available to CSS, allowing web applications to adapt automatically.

Media Query Detects Values
prefers-reduced-motion System reduced motion setting no-preference, reduce
prefers-color-scheme Light or dark theme preference light, dark
prefers-contrast Contrast level preference no-preference, more, less, custom
forced-colors Windows High Contrast Mode none, active

Designer Impact: These media queries are the bridge between system settings and web experiences. As a designer, you specify what should change when each preference is active. Engineering implements the CSS media queries based on your specifications.

Dark Mode Design

Dark mode is the most widely-adopted accessibility preference. It requires intentional color design, not simple color inversion. Key principles include using dark gray backgrounds instead of pure black, off-white text instead of pure white, and replacing shadow-based elevation with borders.

Why not pure black and pure white?

Maximum contrast between #FFFFFF and #000000 can cause halation for users with astigmatism or certain low vision conditions. Bright text appears to glow or blur against very dark backgrounds. Best practice: dark gray backgrounds (#1E1E1E) with off-white text (#E0E0E0).

Semantic design tokens

Use semantic tokens like surface-primary and text-primary instead of hard-coded colors. Map tokens to different values per color scheme so components adapt automatically when the user switches modes.

Designer Impact: Specify dark mode color tokens with verified contrast ratios. Do not assume light mode contrast passes in dark mode. Verify contrast for both color schemes and document shadows that need to be replaced with borders.

Reduced Motion Alternatives

When prefers-reduced-motion: reduce is active, designs should minimize vestibular-triggering motion while maintaining usable feedback. Reduced motion does not mean no motion: it means replacing spatial movement with less triggering alternatives.

Default Animation Reduced Motion Alternative
Content slides in from right (300ms) Content fades in (150ms opacity transition)
Card scales up on hover (1.05x, 200ms) Subtle shadow change, instant
Menu drops down with spring animation Menu appears instantly
Loading spinner rotates continuously Static progress bar or "Loading..." text

Key principle: Spatial motion (slide, scale, bounce, parallax) triggers vestibular responses. Opacity changes (fade in/out) generally do not. Replace spatial motion with opacity or instant transitions when reduced motion is active.

Text Size and Spacing Preferences

Text size preferences work differently from other preferences: they do not use a CSS media query. Instead, when users increase their preferred text size, the browser adjusts the base font size, and CSS rem units scale automatically. Layouts must accommodate text at 200% size without hiding or clipping content (WCAG 1.4.4).

Common text scaling failures

  • Fixed-height containers with overflow: hidden clip scaled text
  • Absolutely-positioned elements overlap when nearby text grows
  • Button labels truncated with ellipsis at large sizes with no access to full text
  • Navigation items that overflow their container horizontally

Designer Impact: Specify rem units for text sizing. Use min-height instead of fixed height. Design layouts that flex when text scales. Document overflow behavior for each component.

Testing Preferences in Chrome DevTools

Chrome DevTools provides a Rendering tab that can emulate CSS media feature preferences without changing system settings. This enables fast, reversible preference testing during design reviews.

How to access preference emulation

  1. Open Chrome DevTools (Cmd+Option+I on Mac, F12 on Windows)
  2. Open the Rendering tab (three-dot menu > More tools > Rendering)
  3. Scroll to "Emulate CSS media feature" section
  4. Toggle: prefers-color-scheme, prefers-reduced-motion, prefers-contrast, forced-colors

Testing strategy: Use DevTools for fast iteration during reviews. Validate with actual system settings before shipping. Test common combinations (dark mode + reduced motion, large text + high contrast), not just individual preferences.
